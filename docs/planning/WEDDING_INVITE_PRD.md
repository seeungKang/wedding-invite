# 모바일 청첩장 PRD (Modern Chic + Christmas Accent)

## 문서 정보
- 문서명: Wedding Invite Product Requirements Document
- 버전: v1.1
- 상태: Draft (운영 관리 시작)
- 최종 수정일: 2026-02-17
- 대상 프로젝트: `wedding-invite`

## 0. 시안 정합성 검토 (예시 스크린샷 기준)
첨부된 스크린샷은 레퍼런스 예시이며, 최종 구현은 본 PRD를 기준으로 한다.

### 0.1 정합 항목
- 모던/시크 톤, 낮은 채도 배경, 그린/레드 포인트 사용
- Hero -> Invitation -> Calendar -> Gallery -> Location -> Accounts -> Footer 흐름
- Location의 지도 프리뷰 + 2개 CTA(네이버/카카오)
- Accounts의 카드형 아코디언 + 복사하기 인터랙션

### 0.2 불일치/보완 필요 항목
1. 날짜 불일치
- 스크린샷 예시: `2026.12.25`
- PRD 기준: `2026.12.19`
- 규칙: 최종 구현은 PRD 날짜를 단일 소스로 사용

2. Guestbook 섹션 노출
- 스크린샷 예시에는 Guestbook 존재
- 기존 PRD에는 명시 부족
- 조치: Guestbook을 선택 기능으로 명시(기본 제외)

3. 계좌 인원 수 불일치
- 스크린샷 예시: 양측 2명 중심으로 보임
- PRD 기준: 총 6명(신랑측 3 + 신부측 3)
- 규칙: 기본 데이터는 6명 구조를 유지, 디자인은 아코디언으로 축약 노출

4. Gallery 상세 정책 구체화 부족
- 썸네일 비율/간격/로딩 실패 처리 기준 추가 필요
- 조치: 5.3과 6장에 명시 보강

## 1. 문서 개요
### 1.1 목적
모바일 중심의 모던/시크한 청첩장을 제공하고, 크리스마스 시즌 분위기를 과하지 않게 포인트로 반영한다. 하객이 정보 확인(일시/장소) → 길찾기(지도) → 축하(계좌복사) → 추억(갤러리)을 빠르게 수행할 수 있도록 설계한다.

### 1.2 타겟
- 모바일로 링크를 전달받아 확인하는 하객(대부분)
- PC에서 확인하는 일부 사용자(회사/집): 중앙 고정 + 양옆 여백 필요

### 1.3 핵심 사용자 목표 (Top Tasks)
1. 결혼식 날짜/시간 확인
2. 장소 확인 및 네이버지도/카카오내비로 바로 이동
3. 축의금 계좌 확인 및 원터치 복사
4. 사진 탐색 및 전체 보기/확대 보기

## 2. 디자인/브랜딩 가이드
### 2.1 톤앤매너
- 모던 + 시크: 절제된 여백, 얇은 라인, 미니멀한 아이콘
- 크리스마스 무드(살짝): 눈송이/리스 등은 장식 정도로 제한, 과한 패턴/반짝 효과 지양
- 전체 베이스: 화이트/그레이/블랙
- 포인트: 그린/레드

### 2.2 컬러 팔레트 (권장)
- Primary Green: `#1F5E3B` ~ `#2B6B46` (텍스트/포인트 라인)
- Accent Red: `#B23A3A` ~ `#C0392B` (포인트, 강조 날짜)
- Neutral
  - White: `#FFFFFF`
  - Gray: `#F5F6F7`
  - Text Gray: `#6B7280`
  - Black: `#111827`

#### 사용 규칙
- 본문/정보 영역은 Neutral 중심
- 그린/레드는 강조(날짜, 구분선, 버튼 아이콘)에만 사용

### 2.3 타이포 (권장)
- 제목: 세리프(고급감) 또는 얇은 산세리프
- 본문: 가독성 우선 산세리프
- 숫자/전화/주소: 가독성 높은 산세리프 + 자간 최소

## 3. 반응형/레이아웃 정책
### 3.1 Mobile First
- 기준 폭: `360~430px`
- 주요 콘텐츠: 단일 컬럼(1열)

### 3.2 PC 뷰 (양 옆 빈 공간)
- 중앙 컨테이너 `max-width: 420~520px` 권장
- 배경: 연한 그레이/오프화이트 또는 은은한 크리스마스 톤
- 좌우 여백 유지 + 콘텐츠 중앙 정렬

### 3.3 공통 UI 룰
- 섹션 간 간격: `24~40px`
- 카드형 박스(계좌/교통/지도 버튼): 라운드 + 얇은 보더
- CTA 버튼 최소 높이: `44px`

## 4. 정보 구조(IA) / 섹션 구성
### 4.1 Hero (인트로)
- 문구: "마음 전하실 곳"/"초대합니다" 등
- 신랑/신부 이름
- 예식 정보 요약(일시/장소)
- 포인트: 상단 얇은 레드 라인 + 그린 타이틀 + 작은 아이콘(선물/홀리 잎)

### 4.2 Calendar (달력)
- 2026년 12월 달력 표시
- 예식일 `2026.12.19 (토)` 강조
- 하단 안내 문구 고정
  - 예: "○○ ♥ ○○의 결혼식은 12월 19일 입니다."

### 4.3 Location (장소/지도)
- 지도 프리뷰
- 홀 이름/주소/전화
- CTA: 네이버 지도, 카카오 내비
- 교통 안내: 지하철/버스 텍스트 리스트

### 4.4 Gallery (갤러리)
- Masonry 그리드
- 전체 보기 기능(별도 화면 또는 모달)
- 썸네일 클릭 시 뷰어(확대/스와이프)

### 4.5 Accounts (마음 전하실 곳)
- 신랑측 3명(신랑 + 혼주 2)
- 신부측 3명(신부 + 혼주 2)
- 총 6인 계좌 복사 가능
- 더보기(아코디언) 형태로 펼침/접기

### 4.6 Footer (선택)
- 연락처/감사문/저작권/공유하기 버튼

### 4.7 Guestbook (선택)
- 목적: 지인 축하 메시지 표시 및 작성
- 기본 정책(MVP): 비활성(제외)
- 확장 정책(Phase 2): 읽기 리스트 + 작성 폼 + 관리자 승인
- 비고: 스크린샷 예시는 템플릿 예시로 간주

## 5. 기능 요구사항
### 5.1 Calendar
#### 목표
예식 날짜를 직관적으로 인지시키기

#### 표시 범위
- 2026년 12월
- 예식일: `2026-12-19 (토)`

#### UX
- 예식일 강조: 빨간색/그린 테두리 등
- 오늘 날짜 강조: 선택
- 하단 문구 고정 노출

#### 옵션
- D-day 표기 (`D-xx`) 선택

### 5.2 Location
#### 목표
길찾기 행동을 최소 클릭으로 연결

#### 지도 프리뷰 정책
- 기본 권장: "미리보기 + 외부 앱 이동"
- 임베드 지도는 스크롤/줌 UX 저하 가능성 고려

#### 표기 정보
- 홀명
- 주소(도로명/지번 중 1개 이상)
- 전화번호(`tel:` 링크)

#### CTA 버튼
- 네이버 지도: 목적지 검색/좌표 기반 링크
- 카카오 내비: 목적지 링크

#### 교통 안내
- SUBWAY: 노선/출구/도보 시간
- BUS: 노선 목록 + 하차 정보

#### 예외/정책
- 외부 앱 미설치 시 웹 또는 스토어 유도
- 링크 실패 시 주소 복사 기능(선택)

### 5.3 Gallery (Masonry + 전체보기 + 뷰어)
#### 목표
감성 전달 + 가벼운 탐색

#### A) 썸네일 리스트 (메인 섹션)
- 레이아웃: Masonry
- 열 수
  - 모바일: 2열 기본
  - 큰 화면: 3열 옵션
- 간격: 8~12px
- 썸네일 radius: 8~12px
- 로딩: 저화질 placeholder -> 로딩 완료 전환
- 실패 처리: 이미지 로드 실패 시 fallback 이미지 또는 회색 블록
- 클릭: 해당 인덱스로 사진 뷰어 진입

#### B) 전체 보기 (Full Gallery)
- 진입 버튼: "전체 보기"
- 형태
  - 안 1: 별도 페이지 라우팅 (`/gallery`)
  - 안 2: 풀스크린 모달
- 내용: masonry 확장 목록
- 추가 로딩: 무한 스크롤 또는 "더보기" 버튼(선택)

#### C) 사진 뷰어 (Photo Viewer)
- 좌/우 스와이프
- 핀치 줌(가능 시) 또는 더블탭 확대(옵션)
- 닫기(X)
- 현재 인덱스 표기 (예: `3 / 24`)
- 접근성
  - PC 키보드 방향키 이동(옵션)
  - 스크린리더용 `alt`/`aria-label`

### 5.4 Accounts (아코디언 + 복사)
#### 목표
계좌 정보를 깔끔하게 숨기고 필요 시 빠르게 복사

#### A) 구성 (총 6명)
- 신랑측
  - 신랑 1명
  - 혼주 2명
- 신부측
  - 신부 1명
  - 혼주 2명

#### B) UI/인터랙션
- 카드 2개(신랑측/신부측)
- 각 카드 내부 항목 리스트
- 기본 접힘 정책(권장)
  - 옵션 1: 첫 항목만 펼침 + 나머지 접힘
  - 옵션 2: 전체 접힘
- 펼침 시 노출
  - 은행명
  - 계좌번호
  - 예금주
  - 복사하기 버튼

#### C) 복사 기능
- 복사 대상
  - 옵션 1: 은행명 + 계좌번호
  - 옵션 2: 계좌번호만
- 동작
  - 탭 -> 클립보드 복사 -> 성공 토스트
- 예외처리
  - iOS 복사 제한 시 대체 안내(길게 눌러 복사 등)

## 6. 공통 정책/비기능 요구사항
### 6.1 성능
- 이미지 최적화(리사이즈/압축, lazy-load)
- 썸네일 저용량, 뷰어 고해상도 점진 로딩
- 첫 화면 LCP 대상(Hero 이미지)은 우선 로드, 나머지 갤러리는 지연 로드
- GitHub Pages 배포 고려: 정적 이미지 포맷 WebP 우선

### 6.2 접근성/사용성
- 버튼 터치 영역 최소 `44px`
- 색상 대비 준수
- 토스트/알림은 `aria-live` 대응
- 아코디언/모달/갤러리 버튼은 `aria-expanded`, `aria-controls`, `aria-label` 적용
- 키보드 포커스 순서와 포커스 링 유지

### 6.3 공유/딥링크 (선택)
- 카카오톡 공유, 링크 복사
- 섹션 앵커 이동 (`#location`, `#gallery`, `#account`)

## 6.4 콘텐츠/데이터 스키마 (운영 필수)
### 기본 이벤트 정보
- `eventDate`: ISO 날짜 (`2026-12-19`)
- `eventTime`: 표시 문자열 (`오후 1시`)
- `venueName`, `venueAddress`, `venuePhone`

### 인물 정보
- `groomName`, `brideName`
- `introMessage[]` (문단 배열)

### 계좌 정보
- `accounts[]`
  - `side`: `groom` | `bride`
  - `role`: `본인` | `혼주`
  - `bankName`, `accountNumber`, `holderName`
  - `copyValue` (복사 대상 문자열)

### 갤러리 정보
- `galleryItems[]`
  - `id`, `thumbSrc`, `fullSrc`, `alt`

## 9. 수용 기준 (Acceptance Criteria)
1. 하객이 10초 내 예식일/장소를 인지할 수 있다.
2. 지도 CTA(네이버/카카오) 클릭 시 외부 길찾기로 연결된다.
3. 계좌 복사 버튼은 성공/실패 피드백(토스트)을 제공한다.
4. 갤러리는 모바일에서 부드럽게 스크롤되고, 뷰어 진입이 가능하다.
5. 모바일/PC 모두 레이아웃 규칙(중앙 고정, 여백, 터치 영역)을 준수한다.
